# Contributing to ForgeCLI

Thank you for your interest in contributing to ForgeCLI! 🚀

ForgeCLI is an open-source DevOps and deployment automation CLI that helps developers deploy, monitor, scale, and manage applications directly from the terminal.

---

## ⭐ Star the Repository

If you find ForgeCLI useful, please consider starring the repository to support the project.

---

## Ways to Contribute

You can contribute by:

* Fixing bugs
* Improving CLI commands
* Enhancing deployment workflows
* Improving Kubernetes and Docker integrations
* Adding monitoring and logging features
* Improving documentation
* Writing tests
* Optimizing performance

---

## Getting Started

### 1. Fork the Repository

Fork the repository to your GitHub account.

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/forgecli.git
cd forgecli
```

### 3. Create a Branch

```bash
git checkout -b feature/your-feature-name
```

---

## Development Guidelines

Please ensure that:

* Code follows the project's style and architecture.
* Changes remain focused on a single issue.
* New features include appropriate documentation.
* Code is readable, maintainable, and tested.
* Security best practices are followed.

---

## Pull Request Guidelines

Before opening a PR:

* Test your changes locally.
* Perform a self-review.
* Reference the related issue.
* Update documentation when necessary.
* Keep PRs focused and easy to review.

Example:

```text
Fixes #123
```

---

## Testing

Before submitting your changes:

* Verify CLI commands function correctly.
* Test deployment workflows when applicable.
* Ensure existing functionality remains unaffected.
* Validate error handling and edge cases.

---

## Reporting Issues

When reporting bugs:

* Provide clear reproduction steps.
* Include relevant logs or screenshots.
* Describe expected and actual behavior.
* Include environment details if applicable.

---

## Community Standards

By contributing to ForgeCLI, you agree to follow the project's Code of Conduct.

Please review `CODE_OF_CONDUCT.md` before participating.

---

## Security

If you discover a security vulnerability:

* Do not disclose it publicly.
* Report it responsibly to the maintainers.
* Include sufficient information for reproduction.

---

Thank you for helping improve ForgeCLI and making DevOps automation more accessible for developers worldwide.
